🔧 Infrastructure Changes
  • Railway Pro + horizontal scaling (multiple replicas)
  • Cloudflare Workers: LINE webhook handler ที่ edge (ลด latency)
  • Cloudflare Queue: buffer OCR requests
  • Neon: read replica สำหรับ dashboard queries
  • แยก cron jobs ออกจาก main API server (เพื่อไม่ให้ cron block traffic)
  • Prometheus + Grafana หรือ Better Stack สำหรับ observability
  • Railway outage risk มีความสำคัญมากขึ้น — ต้องมี fallback plan
⚖️ Compliance & Legal
  • PDPA audit เต็มระบบ
  • ISO 27001 หรือ HITRUST พิจารณา
  • Business Associate Agreement (BAA) กับ Anthropic (ถ้ามี PHI จริง)
  • Data residency: PDPA ต้องการข้อมูลใน TH หรือประเทศที่มี adequacy
Migration Decision Point: ที่ 20k+ users ควรพิจารณาย้าย backend ไป AWS/GCP เพราะ Railway outage 8 ชม. ที่เกิดใน พ.ค. 2026 = พลาด reminder ยา 20k คน × 3 มื้อ = 60k pill misses ความเสี่ยงเชิง reputation และ legal สูง

🔧 Full Migration
