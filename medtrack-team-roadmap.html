🎯 Priority Matrix: จ้างอะไรก่อน–หลัง

🔥 จ้าง/หาเลยตอนนี้ (Phase 0–1)

  • Clinical Advisor (แพทย์) — equity/advisor
  • UX Designer — contractor per project
  • Business/BD Co-founder — equity

💭 พิจารณาช่วง Phase 2 (100–1k users)

  • Full-time Full-stack Developer
  • Customer Success (part-time)
  • Content/Community Manager

⏳ ค่อยจ้างที่ Phase 3+ (1k+ users)

  • Senior Backend Engineer
  • Data Analyst / RWE Specialist
  • Sales Lead (B2B รพ.)

⛔ อย่าเพิ่งจ้าง (Anti-patterns)

  • DevOps engineer (Railway จัดการให้)
  • CFO (ใช้บัญชี freelance พอ)
  • Marketing agency (product ยังไม่พร้อม)
  • Junior devs หลายคน (จัดการยาก)

Phase 0–1 (ตอนนี้–6 เดือน): 3 บทบาทที่ต้องหาก่อน

1. Clinical Advisor (แพทย์)
แพทย์อายุรกรรม หรือ แพทย์เวชศาสตร์ครอบครัว
P1 Advisor Phase 0–1
หน้าที่
• Validate clinical logic ของระบบ reminder (drug interactions, timing rules)
• Review OCR safety nets (Warfarin, Depakine, Florinef edge cases ที่พี่ทำอยู่)
• Endorse ระบบให้ credibility กับ รพ./ผู้ป่วย
• Connect เข้าเครือข่ายแพทย์ + รพ.
ทำไมสำคัญ
พี่เป็นเภสัชกร มี credibility ระดับหนึ่ง แต่การมีแพทย์ร่วมทีมจะเปิดประตูเข้า รพ. และลดความเสี่ยงถ้าโดนถามเรื่อง medical accuracy
Cost: 0.5–1% equity เป็น advisor / หรือ 5–15k/mo retainer part-time / หรือ co-founder 5–15% equity ถ้าเป็น clinical co-founder เต็มตัว
How to find: เริ่มจากเครือข่ายเภสัชกร รพ.รัฐ ที่พี่มี ขอ intro หา residents หรือ fellows ที่สนใจ health tech (มีเยอะขึ้นในไทย)
2. UX/UI Designer
Designer ที่เข้าใจ senior users + LINE ecosystem
P1 Contract Phase 0–1
หน้าที่
• Redesign frontend ทุกหน้า ให้เข้าถึงผู้สูงอายุได้จริง (font ใหญ่, contrast, tap targets)
• Design system ให้ต่อยอดได้ (พี่เริ่ม Style B แล้ว — ต้อง formalize)
• LIFF UX สำหรับ caregiver dashboard
• Onboarding flow (จุดที่ retention มักตาย)
ทำไมสำคัญ
พี่ iterate UI หลายครั้ง (Style A → B, mockup ก่อน code) เพราะรู้ตัวว่าไม่ใช่ designer การจ้าง designer 1 project (~40–60 ชม.) จะประหยัดเวลาพี่ 100+ ชม.
Cost: 40k–80k บาท ต่อ project (redesign หลัก) / หรือ 800–1,500 บาท/ชม. contractor
How to find: Dribbble, Behance, หรือ Thai design community (เช่น Design Thailand, PIXER) — ขอ portfolio ที่มี health/senior focus
3. Business/BD Co-founder
คนที่เอา MedTrack ไปขาย + สร้าง partnership
P2 Co-founder Phase 1–2
หน้าที่
• เอา MedTrack ขายให้ รพ. + ประกัน (พี่มี domain แต่ไม่มีเวลา)
• หา pilot partners + user acquisition
• Grant applications (NIA, Depa, etc.) — พี่ทำเองอยู่ตอนนี้
• Investor relations + fundraising
• Business development + partnerships
ทำไมสำคัญ
Solo founder ที่ทำทั้ง engineering + business = ไม่มีใครทำงานได้ดี พี่มี technical skills แข็ง → หา business partner ที่มี hospital sales experience (นี่คือ pattern ของ Thai health tech ที่ประสบความสำเร็จ: technical + medical/business duo)
Cost: 15–35% equity (ขึ้นกับ commitment และ track record)
Vesting: 4 ปี พร้อม 1 ปี cliff (มาตรฐาน)
Anti-pattern: จ้าง sales เป็น employee ที่ Phase นี้ — ไม่มี product-market fit ยัง sales จะไม่รู้จะขายอะไรและ burnout เร็ว co-founder ที่มี skin in the game จะยอมทน iteration
How to find: เครือข่าย ex-medical rep, pharma marketing, health startup community, Techsauce/Techstars alumni ที่ pivot จาก corporate

เกณฑ์เลือก: ทำ Node.js + Postgres ได้ครบวงจร มีประสบการณ์ debug production issues สำคัญกว่า framework knowledge = "ownership mindset"

💡 ทางเลือกก่อนจ้าง full-time (สำคัญ)

Option A: Contractor Model
จ้าง freelance/agency เป็น project-based สำหรับ UX, mobile testing, content creation
ข้อดี: ไม่มี fixed cost, ยืดหยุ่น
ข้อเสีย: ไม่มี long-term commitment, quality ต่างกันมาก
Option B: Advisor + Equity Model
หา 2–4 advisors ที่มี expertise ต่างกัน (clinical, business, technical, regulatory) — ให้ 0.25–1% equity/คน
ข้อดี: ไม่มี cash outflow, network valuable
ข้อเสีย: commitment ต่ำ (ปกติ 2–4 ชม./เดือน)
Option C: Co-founder Model (แนะนำที่สุดสำหรับ Phase นี้)
หา co-founder 1 คนที่มี business/clinical skills complementary
ข้อดี: shared ownership, ทำงานเต็มที่, no cash out
ข้อเสีย: ต้องแบ่ง equity 15–40%, ต้องหา fit ที่ใช่ (ยาก)
Option D: Internship / Fresh Grad
รับนักศึกษาเภสัช/แพทย์/CS เป็น intern (paid stipend 10–15k/mo)
ข้อดี: ถูก, ได้ potential future hire
ข้อเสีย: ต้องใช้เวลา training มาก (ตรงข้ามกับสิ่งที่พี่ต้องการ)

⛔ Anti-Patterns ที่ Solo Founder มักผิดพลาด

1. จ้าง full-time ก่อนมี revenue — เผา runway หมดใน 6–12 เดือน ก่อนเจอ product-market fit
2. จ้าง junior หลายคนแทน senior 1 คน — พี่จะกลายเป็น manager แทนที่จะเป็น builder เสียเวลามากกว่าเดิม
3. จ้างเพราะเหนื่อย ไม่ใช่เพราะจำเป็น — จะได้ hire ที่ไม่ fit หา contractor ช่วยเฉพาะเรื่องดีกว่าจ้างเต็มตัว
4. ไม่เขียน equity vesting agreement — co-founder ที่ออกก่อนถือ equity ไปมหาศาล เขียน 4-year vesting + 1-year cliff เสมอ
5. จ้างเพื่อน/ญาติเพราะสะดวก — ความสัมพันธ์จะเสีย ถ้า fire ยาก เก็บเพื่อนไว้เป็น advisor/contractor แทน
6. Delegate ก่อน document — ถ้าไม่มี process/docs ชัดเจน คนใหม่จะทำผิด พี่จะ debug 2 เท่าและกลับมาทำเองสุดท้าย
